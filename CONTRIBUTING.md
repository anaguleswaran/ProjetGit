# Guide de contribution

## 1. Récupérer le dépôt original
Fais un pull du repo principal pour récupérer la dernière version du code.

## 2. Forker le projet
Crée un fork du dépôt sur GitHub afin de travailler sur ta propre copie.

## 3. Créer une nouvelle branche
Travaille toujours sur une branche dédiée :
git checkout -b ma-nouvelle-branche
Pour changer de branche ensuite :
git checkout ma-branche
## 4. Ajouter les fichiers modifiés
Après avoir vérifié ton code :
-	Pour tout ajouter :
git add .
-	Pour ajouter un fichier spécifique :
git add chemin/vers/fichier
## 5. Committer tes changements
Utilise un message clair et explicite

## Bonnes pratiques
-	Toujours travailler sur une branche séparée
-	Utiliser des messages de commit clairs et descriptifs
-	Faire régulièrement un pull du repo original pour rester à jour
-	Garder des PR propres, cohérentes et bien expliquées
